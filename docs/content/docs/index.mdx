## 推荐阅读顺序

<Steps>
  <Step>
    先读 [Overview](/docs/getting-started/overview) 和
    [Quickstart](/docs/getting-started/quickstart)，知道系统怎么启动、成功信号是什么。
  </Step>
  <Step>
    再读 [哲学理念](/docs/concepts/philosophy) 与
    [系统架构](/docs/concepts/architecture)，建立“为什么要这样设计”的判断框架。
  </Step>
  <Step>
    根据角色继续深入：
    Console 面向使用者，Developer 面向仓库开发者，Reference 面向查契约和运行时细节的人。
  </Step>
</Steps>
